# MoSCoW

## Introdução

- Para priorizar os requisitos estamos utilizando a técnica de MoSCoW, que se baseia nos seguintes aspectos:
  - Must: Deve ter
  - Shoud: Deveria ter
  - Could: Pode ter
  - Would: Deveria ter
- A prioridade é de **Would** para menos importante até **Must** para mais importante.
